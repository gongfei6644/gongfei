字段类型：

1、CharFiled 提供一种字符串字段类型。这是一种比较短的字符的存储类型，如果你需要存储大量的字符串，可以使用TextField。

xxxxxxxxxx
class CharField(max_length=None, **options):

示意：

xxxxxxxxxx
models.CharField(max_length=100,verbose_name=u'标题')

2、TextField 提供一种长的字符串存储的类型。

xxxxxxxxxx
class TextField(**options)
A large text field. The default form widget for this field is a Textarea.

3、DateTimeField 提供一种日期字段类型。

xxxxxxxxxx
class DateTimeField(auto_now=False, auto_now_add=False, **options)
#A date and time, represented in Python by a datetime.datetime instance.

示意：

xxxxxxxxxx
models.DateField(default=datetime.now,verbose_name=u'添加时间')

4、EmailField 提供了一中邮箱字段的验证方式。

xxxxxxxxxx
lass EmailField(max_length=254, **options)
A CharField that checks that the value is a valid email address using EmailValidator.

示意：

xxxxxxxxxx
models.EmailField(max_length=50, verbose_name=u'邮箱')

5、ImageField 提供一种图片的存储方式的字段，他会将字段转化为存储图片的地址。

xxxxxxxxxx
class ImageField(upload_to=None, height_field=None, width_field=None, max_length=100, **options)

示意

xxxxxxxxxx
models.ImageField(upload_to='image/%Y/%m',default='image/defaule.png')

6、IntegerField 提供一中数字类型的字段。

xxxxxxxxxx
class IntegerField(**options)

示意：

xxxxxxxxxx
models.IntegerField(default=100, verbose_name=u"顺序")

7、URLFiled 提供给了一种url的字段类型，默认长度为200。

xxxxxxxxxx
class URLField(max_length=200, **options)

示意：

xxxxxxxxxx
models.URLField(max_length=200,verbose_name=u'访问地址')

8、Foreignkey 提供一种外键的链接，第一个参数是外键的链接的表明。

示意：

xxxxxxxxxx
models.ForeignKey(CityDict, verbose_name=u'所在城市')

9、BooleanField 提供一种布尔字段类型。

xxxxxxxxxx
class BooleanField(**options)
A true/false field.

示意：

xxxxxxxxxx
# 例如消息时候为已读
models.BooleanField(default=False,verbose_name=u'是否已读')

10、FileField 文件类型提供上传下载文件的链接：

xxxxxxxxxx
class FileField(upload_to=None,max_length=254,**options)

示意：

xxxxxxxxxx
models.FileField(upload_to='course/resource/%Y/%m',verbose_name=u'文件资源')

 

字段的类型选项：

用法：

xxxxxxxxxx
class userinfo(models.Model):
    name = models.Charfield(字段选项类型)

1、verbose_name 

xxxxxxxxxx
    A human-readable name for the field. If the verbose name isn’t given, Django will automatically create it using the field’s attribute name, converting underscores to spaces. See Verbose field names.

提供一种我们人类可阅读的方式，例如我们可以定义为中文字符。eg:verbose_name = u'用户名’

2、default

xxxxxxxxxx
    The default value for the field. This can be a value or a callable object. If callable it will be called every time a new object is created.

提供了我们创建字段的时候的默认值。default = u'二蛋'

3、choices

xxxxxxxxxx
    A sequence consisting itself of iterables of exactly two items (e.g. [(A, B), (A, B) ...]) to use as choices for this field. If choices are given, they’re enforced by model validation and the default form widget will be a select box with these choices instead of the standard text field.

为我们提供了一中可以选择类型的数据。一般是以键值对的形式对应。

xxxxxxxxxx
YEAR_IN_SCHOOL_CHOICES = [
('FR', 'Freshman'),
('SO', 'Sophomore'),
('JR', 'Junior'),
('SR', 'Senior'),
]
# 亦可以将列表换乘元祖
student_type = models.Charfiled(stutyoe=YEAR_IN_SCHOOL_CHOICES)

4、max_length

提供一种指定字段类型长度的方法，eg:max_length = 100,

5、blank

xxxxxxxxxx
If True, the field is allowed to be blank. Default is False.if True, the field is allowed to be blank. Default is False.

提供了一种是否该字段可以为空的选择。eg:blank=True/Flase ，默认为False.

6、null

xxxxxxxxxx
If True, Django will store empty values as NULL in the database. Default is False.

如果是True的话，在数据库中，Django会将我们设置的字段值设置为null，默认为False.

创建models

1) 创建用户app:users：

在创建用户的app的时候Django已经为我们提供了auth-users表，我们很多的条件他都没有办法满足，例如：昵称、生日、电话、性别、地址等。所以我们将在Django提供的user基础上去重写他的users类。

用户信息表：UserProfile

​x
from django.contrib.auth.models import AbstractUser
​
class UserProfile(AbstractUser):
    # 昵称 
    nick_name = models.CharField(max_length=50,verbose_name=u'昵称'，default='')
    # 生日 birthday 字符串类型   允许为空
    # 性别 gender 字符串类型  最大长度为6 可以供选择的类型 默认为男
    # 地址 address 字符串类型  最大长度为100，默认为空
    # 电话 phone 字符串类型  最大长度为11，默认为空
    # 头像 image 图片类型   上传的地址为'image/%Y/%m' 默认为’image/default.png‘ 最大长度为100
